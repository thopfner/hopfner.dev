# Coding Agent Prompt

You are executing a parity program, not a feature sprint.

The audit conclusion is:

1. formatting parity is mostly strong, but a few cross-cutting gaps remain
2. content-control parity is still materially incomplete
3. the worst remaining issues are not hidden edge cases; they are high-visibility section workflows
4. the only safe way to finish this is one phase at a time

Execution rules:

1. execute phases in order
2. stop after each phase and report
3. do not widen scope
4. if a phase cannot be completed cleanly, stop and report the exact blocker

Anti-drift rules:

1. do not use capability-matrix parity as a substitute for real editor parity
2. do not leave a section in “partial parity” if the phase explicitly targets it
3. do not degrade the current visual-editor shell to force parity
4. do not invent new CMS contracts or persistence models

Required reporting after each phase:

1. exact files changed
2. exact controls/features brought to parity
3. exact tests added or upgraded
4. exact live QA scenarios performed
5. exact command results for test and build
