# Visual Editor v8 UX And Product-Polish Brief

This bundle is the next focused brief for the admin visual editor.

It is built on the current QA read:

- the implementation is materially improved
- the visual editor is now credible
- the remaining problems are mostly UX, state-management precision, and product-finish issues

## Overall Grade

Current coding-agent grade: `B+`

Why:

- architecture is moving in the right direction
- tests pass
- build passes
- the editor now feels real
- but it is still not at the “elite SaaS product” bar because a few high-friction UX issues remain in core authoring flows

## Primary v8 Goals

1. fix large-text editing so the editable surface is actually usable
2. fix false dirty-state prompts so only real changes count as unsaved
3. upgrade shell UX so the editor feels faster, clearer, and more premium at page scale
4. raise the product finish around structure, navigation, and editing context

## Important Positioning

v8 is not another broad architecture pass.

It is a product-finish pass around the editor experience itself.
